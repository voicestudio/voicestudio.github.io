YouTube API 등록 가이드

본인 Google Cloud 프로젝트 연결하기

YouTube 업로드 쿼터(일일 10,000 units)를 본인 계정에 귀속시키기 위해 Google Cloud에서 client_secrets.json을 만들어 앱에 등록합니다. 약 7~10분 소요.

⚙️ 2026년 최신 UI (Google 인증 플랫폼) 기준 · 화면 이미지 포함
왜 이 작업이 필요한가요?
YouTube API 쿼터는 OAuth 클라이언트가 속한 Google Cloud 프로젝트 단위로 부과됩니다. 모든 사용자가 하나의 프로젝트를 공유하면 쿼터가 금방 소진되기 때문에, 각자 본인의 프로젝트를 만들어 연결해야 안정적으로 업로드할 수 있습니다.
1

Google Cloud Console 접속 & 프로젝트 생성

  1. console.cloud.google.com 접속 (본인 Google 계정으로 로그인)
  2. 첫 접속 시 약관 동의 화면 → 국가 선택, 서비스 약관 체크 후 동의
  3. 좌측 상단 프로젝트 선택 드롭다운 클릭
  4. 팝업에서 "새 프로젝트" 클릭
  5. 프로젝트 이름 입력 (예: my-cozy-uploader) → "만들기"
  6. 생성 완료 알림 뜨면 상단 드롭다운에서 방금 만든 프로젝트 선택
💡 팁: 상단 "무료 체험판 상태: ₩453,008 크레딧" 배너는 "닫기" 무시. YouTube API 는 무료 쿼터 사용이라 크레딧 안 씀.
2

YouTube Data API v3 활성화

  1. 왼쪽 햄버거 메뉴(☰) → "API 및 서비스" → "라이브러리"
  2. 검색창에 YouTube Data API v3 입력 후 결과 클릭
  3. "사용" 버튼 클릭 → 활성화 완료
직접 이동 링크
3

Google 인증 플랫폼 시작하기

2026년부터 Google 이 OAuth 동의 화면 UI를 "Google 인증 플랫폼"으로 개편했습니다. 왼쪽 메뉴가 개요 브랜딩 대상 클라이언트 형태입니다.

3-1. 시작하기 화면

  1. 왼쪽 메뉴 → "API 및 서비스" → "OAuth 동의 화면"
  2. "Google 인증 플랫폼이 아직 구성되지 않음" 화면 → 파란 "시작하기" 클릭
4

앱 정보 마법사 (4단계)

4-1. 앱 정보 (1/4)

  • 앱 이름: My Cozy Uploader (또는 원하는 이름)
    • ⚠️ "Google" / "YouTube" 같은 브랜드명 사용 금지
  • 사용자 지원 이메일: 드롭다운에서 본인 Google 이메일 선택
  • "다음"

4-2. 대상 (2/4)

  • 외부(External) 선택
  • Internal 은 Google Workspace 계정 전용이라 일반 사용자는 선택 불가
  • "다음"

4-3. 연락처 정보 (3/4)

  • 이메일 주소: 본인 Google 이메일 입력
  • Google 이 정책 변경 시 알림 보냄
  • "다음"

4-4. 약관 동의 (4/4)

  • "Google API 서비스: 사용자 데이터 정책" 체크
  • "완료" 또는 "만들기"
5

테스트 사용자 추가 (⭐ 필수)

이 단계를 빠뜨리면 본인 계정으로도 로그인 불가.

  1. 왼쪽 메뉴에서 대상 탭 클릭
  2. "테스트 사용자" 섹션 찾기
  3. "+ 사용자 추가" 버튼 클릭
  4. 본인 Google 이메일 입력 (YouTube 채널 소유 계정)
    • 여러 채널이면 각 채널 소유 이메일 모두 추가
  5. "저장"
⛔ "앱 게시" 절대 금지.
"테스트 중" 상태 유지 → 본인/테스트 사용자만 쓸 수 있고 만료 없음. "게시" 누르면 Google 심사 필요.
6

OAuth 클라이언트 ID 생성 (핵심)

  1. 왼쪽 메뉴에서 클라이언트 탭 클릭
  2. 상단 "+ 클라이언트 만들기" 버튼
  3. 애플리케이션 유형: "데스크톱 앱" 선택
    • ⚠️ "웹 애플리케이션" 아님
  4. 이름: Cozy Uploader Desktop
  5. "만들기"
  6. 생성 완료 팝업에서 "JSON 다운로드" 클릭
  7. client_secret_xxxx.apps.googleusercontent.com.json 파일 저장
✅ 이 파일 절대 공유 금지. 본인 프로젝트의 인증 열쇠입니다.
7

Cozy Uploader 에 파일 등록

  1. Cozy Uploader 앱 실행 → 로그인
  2. 왼쪽 사이드바 → "설정" 탭
  3. 스크롤 → "YouTube API 인증 파일" 항목
  4. "파일 등록" 클릭 → 다운로드한 JSON 선택
  5. "등록 완료" 메시지 확인
  6. "YouTube 채널 > 계정 연결" 버튼 클릭
  7. 브라우저 열리면 본인 Google 계정으로 로그인
8

"확인되지 않은 앱" 경고 통과

OAuth 로그인 시 "Google에서 이 앱을 확인하지 않았습니다" 경고가 뜹니다. 본인 프로젝트이므로 안전합니다.

  1. 경고 화면 → "고급" 또는 "Advanced" 링크 클릭
  2. 하단 "(앱 이름)(안전하지 않음)(으)로 이동" 클릭
  3. 권한 요청 → "허용" 클릭
  4. "인증 완료! 탭을 닫으세요" 메시지 확인 → 탭 닫기
  5. 앱으로 돌아가서 채널 이름 표시 확인
경고가 뜨는 이유: 본인이 만든 앱을 Google 심사받지 않았기 때문. 본인 프로젝트에서 본인 계정 로그인은 안전.

자주 묻는 질문

Q. 유료인가요?
Google Cloud 자체는 무료로 시작. YouTube Data API 는 일일 10,000 units 무료 쿼터 제공. 영상 1개 업로드당 약 1,700 units 이므로 하루 5~6개까지 무료 업로드 가능. 처음 가입 시 제공되는 ₩450,000 무료 크레딧은 YouTube API 호출에 사용되지 않습니다.
Q. 하루 10개 이상 업로드하고 싶어요.
Google Cloud Console → "API 및 서비스 > 할당량" 에서 확장 신청 가능. Professional 등급 (채널 6개 × 6영상 = 36개/일) 사용하려면 확장 필요. 심사 1~2개월.
Q. 왼쪽 메뉴가 예전 가이드 화면과 달라요.
2026년 Google 이 OAuth UI를 "Google Auth Platform" 으로 개편. 왼쪽 메뉴가 개요 브랜딩 대상 클라이언트 형태면 신버전. "사용자 인증 정보" · "OAuth 동의 화면" 형태면 구버전. 이 가이드는 신버전 기준.
Q. 본인 확인 없이 사용하고 싶어요.
대상 탭을 "테스트 중" 상태로 유지하고 본인을 테스트 사용자로 추가하면 됨. "앱 게시" 누를 때만 Google 심사 필요.
Q. 여러 YouTube 채널 관리 가능?
네. client_secrets.json 하나로 여러 채널 연결 가능. 다른 Google 계정 채널은 해당 이메일도 대상 탭 테스트 사용자에 추가.
Q. 파일 해킹되면?
클라이언트 탭에서 해당 클라이언트 삭제 후 새로 생성 → 즉시 무효화. Cozy Uploader 는 이 파일을 Windows Credential Manager 로 암호화 저장.
Q. 영어로만 나와요.
Google Cloud Console 우측 상단 프로필 → 설정 → 언어에서 "한국어" 로 변경. 영어 그대로 진행해도 동일.
🎉 등록 완료 시, Cozy Uploader 로 돌아가서 수동 업로드 / 완전자동화 사용 가능. 쿼터 부족 시 "API 및 서비스 > 할당량" 에서 사용량 확인.
