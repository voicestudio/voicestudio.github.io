YouTube API 등록 가이드

본인 Google Cloud 프로젝트 연결하기

YouTube 업로드 쿼터(일일 10,000 units)를 본인 계정에 귀속시키기 위해 Google Cloud에서 client_secrets.json을 만들어 앱에 등록합니다. 약 7~10분 소요.

⚙️ 2026년 최신 UI (Google 인증 플랫폼 = Google Auth Platform) 기준
왜 이 작업이 필요한가요?
YouTube API 쿼터는 OAuth 클라이언트가 속한 Google Cloud 프로젝트 단위로 부과됩니다. 모든 사용자가 하나의 프로젝트를 공유하면 쿼터가 금방 소진되기 때문에, 각자 본인의 프로젝트를 만들어 연결해야 안정적으로 업로드할 수 있습니다.
1

Google Cloud Console 접속 & 프로젝트 생성

  1. console.cloud.google.com 에 접속 (본인 Google 계정으로 로그인)
  2. 첫 접속 시 약관 동의 화면이 나올 수 있음 → 국가 선택, 서비스 약관 체크 후 동의
  3. 상단 프로젝트 선택 드롭다운(좌측 상단, 보통 "프로젝트 선택" 또는 기본 프로젝트명 표시) 클릭
  4. 팝업에서 "새 프로젝트" 클릭
  5. 프로젝트 이름 입력 (예: my-cozy-uploader) → "만들기" 클릭
  6. 생성 완료 알림(우측 상단 종 아이콘)이 뜨면 상단 드롭다운에서 방금 만든 프로젝트 선택
💡 팁: 상단에 "무료 체험판 상태: ₩453,008 크레딧 및 90일 남았습니다" 배너가 뜰 수 있음 — YouTube API는 무료 쿼터 사용이라 무시해도 됩니다. 오른쪽 "닫기" 클릭.
2

YouTube Data API v3 활성화

  1. 왼쪽 햄버거 메뉴(☰) → "API 및 서비스" → "라이브러리"
  2. 검색창에 YouTube Data API v3 입력 후 결과 클릭
  3. "사용" 버튼 클릭 → 활성화 완료 페이지로 이동
직접 이동 링크
3

Google 인증 플랫폼 시작하기 (OAuth 동의 화면)

2026년부터 Google Cloud Console의 OAuth 동의 화면 UI가 "Google 인증 플랫폼"(Google Auth Platform)으로 개편되었습니다. 왼쪽 메뉴 구조도 바뀌었습니다.

3-1. 시작하기 화면

  1. 왼쪽 햄버거 메뉴 → "API 및 서비스" → "OAuth 동의 화면"
  2. 또는 주소창에 직접 입력: console.cloud.google.com/apis/credentials/consent
  3. "Google 인증 플랫폼이 아직 구성되지 않음" 화면 → 한가운데 파란 "시작하기" 버튼 클릭

3-2. 앱 정보 입력 (마법사 1/4)

  • 앱 이름: My Cozy Uploader (또는 원하는 이름)
    • ⚠️ "Google" / "YouTube" 같은 브랜드명은 피하세요 (규정 위반)
  • 사용자 지원 이메일: 드롭다운에서 본인 Google 이메일 선택
  • "다음" 클릭

3-3. 대상 (Audience) 선택 (마법사 2/4)

  • 외부(External) 선택 (Internal 은 Google Workspace 계정 전용)
  • "다음" 클릭

3-4. 연락처 정보 (마법사 3/4)

  • 이메일 주소: 본인 Google 이메일 입력 (Google 이 정책 변경 등을 알릴 때 사용)
  • "다음" 클릭

3-5. 약관 동의 (마법사 4/4)

  • "Google API 서비스: 사용자 데이터 정책" 체크박스 동의
  • "완료" 또는 "만들기" 클릭

생성 완료 화면이 뜨면 "대시보드로 이동" 또는 바로 다음 탭으로 넘어갑니다.

4

테스트 사용자 추가 (⭐ 필수)

이 단계를 빠뜨리면 본인 계정으로도 로그인 불가합니다.

  1. 왼쪽 메뉴에서 대상 탭 클릭 (Audience)
  2. "테스트 사용자" 또는 "Test users" 섹션 찾기
  3. "+ 사용자 추가" (또는 "+ ADD USERS") 버튼 클릭
  4. 팝업에 본인 Google 이메일 입력 (YouTube 채널을 소유한 계정)
    • 여러 채널을 쓸 거면 각 채널 소유 이메일 모두 추가
  5. "저장" 클릭
⛔ 절대 "앱 게시(Publish)" 버튼을 누르지 마세요.
"테스트 중" 상태로 두면 본인 + 테스트 사용자만 쓸 수 있고 만료 시한 없음. "게시" 누르면 Google 심사 들어가고, 통과 안 되면 앱을 못 씁니다. 본인 개인용이면 테스트 모드가 정답.
5

OAuth 클라이언트 ID 생성 (핵심 단계)

  1. 왼쪽 메뉴에서 클라이언트 탭 클릭 (Clients)
    • 구버전 UI를 쓰는 경우: "API 및 서비스" → "사용자 인증 정보"
  2. 상단 "+ 클라이언트 만들기" 또는 "+ CREATE CLIENT" 버튼 클릭
  3. 애플리케이션 유형: "데스크톱 앱" (Desktop app) 선택
    • ⚠️ "웹 애플리케이션"이 아님. Cozy Uploader는 데스크톱 앱
  4. 이름: Cozy Uploader Desktop (자유)
  5. "만들기" 클릭
  6. 생성 완료 팝업에 클라이언트 ID와 Secret 표시됨 → 우측의 "JSON 다운로드" 또는 다운로드 아이콘 클릭
  7. client_secret_xxxx.apps.googleusercontent.com.json 파일 저장 (보통 다운로드 폴더)
✅ 다운로드한 파일은 절대 공유/깃허브 업로드하지 마세요. 이 파일이 곧 본인 프로젝트의 인증 열쇠입니다.
6

Cozy Uploader 에 파일 등록

  1. Cozy Uploader 앱 실행 → 로그인
  2. 왼쪽 사이드바 → "설정" 탭 클릭
  3. 스크롤 내려서 "YouTube API 인증 파일" 항목 찾기
  4. "파일 등록" 버튼 클릭 → 방금 다운로드한 client_secret_xxxx.json 선택
  5. "등록 완료" 메시지 확인
  6. 동일 설정 탭에서 "YouTube 채널 > 계정 연결" 버튼 클릭
  7. 브라우저가 자동으로 열리면 본인 Google 계정으로 로그인
7

"확인되지 않은 앱" 경고 통과

OAuth 로그인 시 "Google에서 이 앱을 확인하지 않았습니다" 경고 화면이 뜹니다. 본인이 만든 프로젝트이므로 안전합니다.

  1. 경고 화면에서 "고급" 또는 "Advanced" 링크 클릭
  2. 하단의 "(앱 이름)(안전하지 않음)(으)로 이동" 클릭
  3. 권한 요청 화면에서 "허용" 클릭 (YouTube 업로드 · 관리 · 분석 권한)
  4. "인증 완료! 이 탭을 닫고 앱으로 돌아가세요" 메시지 확인 → 브라우저 탭 닫기
  5. Cozy Uploader 로 돌아와 채널 이름이 표시되는지 확인
경고가 뜨는 이유: 본인이 만든 앱을 Google 에 심사받지 않았기 때문입니다 (심사받으려면 복잡한 절차 필요). 본인 프로젝트에서 본인 계정으로 로그인하는 경우 이 경고를 통과해도 안전합니다.

자주 묻는 질문

Q. 유료인가요?
Google Cloud 자체는 무료로 시작할 수 있고, YouTube Data API 는 일일 10,000 units 의 무료 쿼터를 제공합니다. 일반적으로 영상 1개 업로드당 약 1,700 units 를 사용하므로 하루 5~6개까지 무료로 업로드 가능합니다. 처음 가입 시 제공되는 ₩450,000 무료 크레딧은 YouTube API 호출에 사용되지 않습니다.
Q. 하루 10개 이상 업로드하고 싶어요.
Google Cloud Console 에서 쿼터 확장을 신청할 수 있습니다 (API 및 서비스 > 할당량). Professional 등급 (채널 6개 × 6영상 = 36개/일)을 사용하려면 확장 신청이 필요하며, 심사에 1~2개월 소요될 수 있습니다.
Q. 왼쪽 메뉴가 예전 가이드 화면과 달라요.
2026년에 Google 이 OAuth 설정 UI를 "Google Auth Platform" (Google 인증 플랫폼) 으로 개편했습니다. 왼쪽 메뉴가 개요 브랜딩 대상 클라이언트 데이터 액세스 형태라면 신버전, "사용자 인증 정보" · "OAuth 동의 화면" 형태라면 구버전입니다. 이 가이드는 신버전 기준이며, 구버전에서도 같은 작업을 조금 다른 위치에서 할 수 있습니다.
Q. 본인 확인 없이 사용하고 싶어요.
대상 탭을 "테스트 중" 상태로 두고 본인을 테스트 사용자로 추가하면 별도 확인 없이 사용 가능합니다. "앱 게시" 를 눌러야만 Google 심사가 필요해집니다. 본인/지인만 쓰는 앱은 굳이 게시할 필요 없습니다.
Q. 여러 YouTube 채널을 하나의 프로젝트로 관리할 수 있나요?
네. client_secrets.json 하나로 여러 채널(같은 Google 계정 내)을 OAuth 로 연결할 수 있습니다. 다른 Google 계정의 채널을 쓰려면 해당 계정의 이메일도 대상 탭의 테스트 사용자에 추가하세요. Cozy Uploader 앱의 "채널 프로필" 기능으로 여러 채널을 전환할 수 있습니다.
Q. 파일이 해킹당하면 어떻게 되나요?
클라이언트 탭에서 해당 OAuth 클라이언트를 삭제하고 새로 만들면 즉시 무효화됩니다. 또한 Cozy Uploader 앱은 이 파일을 Windows Credential Manager 로 암호화해서 저장합니다.
Q. "시작하기" 가 아닌 "GET STARTED" 등 영어로 나와요.
Google Cloud Console 우측 상단 프로필 아이콘 → 설정 → 언어에서 "한국어" 로 바꿀 수 있습니다. 영어 그대로 진행해도 동일합니다 (이 가이드의 영문 버튼명과 비교).
🎉 등록이 완료되었다면, Cozy Uploader 로 돌아가서 수동 업로드 또는 완전자동화 기능을 사용해보세요. 쿼터 부족으로 업로드가 실패할 때는 Google Cloud Console 의 "API 및 서비스 > 할당량" 에서 현재 사용량을 확인할 수 있습니다.
